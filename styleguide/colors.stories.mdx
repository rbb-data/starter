## chart colors

### primary

These blues are the default colors for a simple chart with a single category.  
Use this if the theme don't suggest otherwise (e.g. green for parks)

**Don't use the different shades for multiple categories!**  
There are miltiple shades so you can adjust to a background 
or use a darker color for thinner lines.  
A common pattern would be to use a darker blue as an outline for a shape filled with a lighter one.

### category colors

These colors are to encode categories without implicit order.

By default use the colors from the second row. 
If you need to highlight or fade out someting you can use
the darker or lighter vaersion of the colors.  
Don't use different shades of the same color for different categories.

Try to choose colors that match your category: Like blue for winter or green for nature.  
Don't use the same color for different categories in one article.  
If you don't know which colors to pick just go from left to right.

### gradients

These gradients are created with chroma js.  
You can copy the colors to the [chroma-js color scale helper](https://gka.github.io/palettes/#/11|s|fa9d1e,dfded2,3fbaed|ffffe0,ff005e,93003a|0|0)if you need
nore or less colors in a scale.

#### diverging

Use for two dimesional data. Exeamples for when to use a diverging scale:
  - above see level / below see level (beige: is see level)
  - above average / below average (beige: is average)
  - more people are leaving / more people are arriving (beige: the same number of people are leaving and arriving) 

<ColorPalette>
  <ColorItem title="changes" subtitle="from increase over unchanged to decrease" colors={Chroma.scale(['#fa9d1e', colors.beige, colors.lightBlue]).colors(13)} />
  <ColorItem title="darkCyan | cyan | beige | honey | darkHoney" subtitle="diverging scale" colors={Chroma.scale(['#00685c', '#008373', colors.beige, '#fad700', adjustHue(-2, '#ebb600')]).mode('lab').colors(13)} />

</ColorPalette>
