# Colors

## color use

Good basic advice for color use: https://blog.datawrapper.de/colors/

**Don't use the different shades for multiple categories!**  
There are multiple shades so you can adjust to a background or use a darker color for thinner lines.  
A common pattern would be to use a darker blue as an outline for a shape filled with a lighter one.

## chart colors

### primary

Blue is the default color for a simple chart with a single category.  
Use this if the theme does not suggest otherwise (e.g. green for parks)

### category colors

Try to choose colors that match your category: Like cyan for winter and yellow for summer or green for yes and magenta for no.  
Try not to use the same color for different categories in one article.  
If you don't know which colors to pick just go from left to right.

### gradients

#### linear

Use for one dimesional data.  
The first color always encodes 0. So depending on your data (e.g. if your first chunk is something like: _from 0 - 5_) you might want to start with the second color.

<GradientBoxes title='cyan-blue-scale' scale={gradients.linear.cyanBlue} />
<GradientBoxes

/>
<GradientBoxes

/>
<GradientBoxes
  title='NO2 scale'
  description='NO2 values from 0 to 100'
  scale={gradients.linear.no2}
/>

#### diverging

Diverging scales should be used with two dimesional data or when da data fluctuates around a neutral value like 0.

**Exeamples for when to use a diverging scale:**

- voted for us republicnas / voted for us democrats (beige: both parties got the same amount of votes)
- above see level / below see level (beige: is see level)
- above average / below average (beige: is average)
- more people are leaving / more people are arriving (beige: the same number of people are leaving and arriving)
