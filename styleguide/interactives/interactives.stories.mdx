# click red button first, ask questions later

The idea is to build interactives that have one primary action.  
For example moving a range slider or clicking a play button.  
Normally this action can be encoded with something like a red button.

Maybe after the primary action, the user can replay the animation, select or highlight a
specific data point or filter the data.  
But maybe the primary action is all we need.

## Slider

A different primary action can be a slider.  
So a start screen (or initial data point) is already selected
and the user can slide through the data points.

If she is interested in a specific data point it can be selected
in the map or overview below or above
